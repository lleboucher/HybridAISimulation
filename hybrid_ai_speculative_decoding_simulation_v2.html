When the network enters the generation loop

A small model on the device drafts a batch of tokens. A large model in the cloud or edge verifies the whole candidate block in one forward pass. If RTT and jitter exceed the budget, the network becomes the slow stage in the inference graph.

Bottleneck formula

L ≈ V
current L ≈ one target forward pass = 90.0 ms
V = verifier block pass = 90.0 ms
A = accepted tokens after block verification
hybrid_cost = (D + R + V) / A
network budget R* = A x L - D - V

The network becomes a bottleneck when actual or p99 RTT exceeds R*. In this model, the large verifier scores the proposed block in one forward pass; the accept/reject decisions appear together at the end of that pass.
